# 2.6.3 关系运算符

关系运算符是指对两个数字数据大小进行判断的运算符，关系运算符包含的内容如表2.4所示。

表2.4 关系运算符

关系运算符 含义
> 大于
< 小于
>= 大于或等于
<= 小于或等于
=== 等于
!== 不等于

关系运算符运算的数据是数字类型的，不过其结果是布尔类型的，返回值是真（true）或假（false），即成立或不成立。在代码中的表示形式如下。

> 34 > 79
< false
1
2

这里判断34是否大于79，其结果是false，表示34不大于79。

注意

这里每一行的第一个字符，即“34>79”和“false”前面的字符，不是关系运算符，而是输入提示符和输出提示符。

大于或小于的关系运算符都都比较好理解，而对于等于关系运算符，可能需要我们额外解释一下。

前面我们在给变量赋值是使用的是单等号，其含义是将一个数据赋值给变量，所以不能像在数学应用中那样使用单等号来判断两个条件是否相等。

在JavaScript中使用三等好来判断两个条件是否相等。使用等于关系运算符时，只有两边的条件相等时，结果才为真，否则就是假。注意等于关系运算符并不只是判断数字类型的数据，也可以判断字符串类型以及布尔类型的数据，所以在两边条件相等的情况下，两者数据类型一样，同时具体是数据也一样。如果比较的是两个不同类型的数据，则其结果总是false。假如我们比较一个字符串类型的数据11和一个数字类型的数据11，虽然两者的值一样，但因为数据类型不同，所以返回的值还是false。在控制台中的效果如下。

> var numString = "11"
< undefined
> var numNum = 11
< undefined
> numNum === numString
< false

这里我们定义了一个字符串类型的变量numString以及一个数字类型的变量numNum，它们的值都是11，当我们用等于关，系运算符比较两者时，返回的结果是false。

另外在JavaScript中还有一种双等号的操作符，这种操作符也能够判断两个条件是否相等，不过它不会判断两个条件的数据类型，在上面那种情况中，虽然一个是数字类型，一个是字符串类型，但如果使用双等号操作符进行判断返回的值就是真。

> numNum == numString
< true
1
2

所以在使用双等号时一定要小心，因为JavaScript中在使用“==”进行比较时，首先会尝试把两个值当作相同的类型，然后再进行比较。我们来看下面这个例子。

> false == 0
< true
> "false" == false
< false
> 45 == true
< false
> 1 == true
< true

通常，在JavaScript中尝试转换时，会将布尔类型的数据转换为数字类型的数据，数字形式的字符串也会转化为数字类型的数据。布尔类型的数据true就是1，而false就是0，所以在上面的例子中，判断false是否等于0时，返回的结果是真；而在判断1是否等于true时，返回的值也是真，最后剩下的两个进行比较，返回值都是假。

因此，在JavaScript中进行比较时，建议大家还是使用三等号的等于关系运算符，会更加严谨一些。
